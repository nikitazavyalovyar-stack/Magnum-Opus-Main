# 🧠 Техническая архитектура (Magnum Opus Technical Architecture)

## 📖 Описание

**Техническая архитектура** — это структура проекта, которая позволяет масштабироваться до 10,000 игроков. Вдохновлена Star Citizen, EVE Online и Destiny 2, но с уникальными особенностями для космической MMO.

**Философия:** "Простота — ключ к масштабируемости."

---

## ⚙️ Основные принципы

### 1. Модульная архитектура
Проект разделён на **модули**, которые могут работать независимо.

**Модули:**
- **Core** — основные системы (логин, сохранение)
- **Gameplay** — игровые механики (строительство, бои)
- **Economy** — экономика (валюты, биржа)
- **UI** — интерфейс (HUD, меню)
- **Networking** — сеть (Netcode for GameObjects)
- **Database** — база данных (хранение игроков)

**Преимущества:**
- Легко добавлять новые механики
- Просто тестировать и отлаживать
- Нет циклических зависимостей

### 2. Слойная архитектура
Проект разделён на **слои**, которые взаимодействуют через чёткие интерфейсы.

**Слои:**
- **Presentation** — UI и визуализация
- **Application** — бизнес-логика
- **Domain** — сущности и правила
- **Infrastructure** — данные и сетевые вызовы

**Пример:**
- Presentation: HUD, меню
- Application: логика строительства
- Domain: блоки, стены, двери
- Infrastructure: сохранение в базу данных

### 3. Система событий (Event System)
Все модули общаются через **события**, а не напрямую.

**Пример:**
- Player строит стену → событие `WallBuiltEvent`
- Система экономики слушает событие → списывает ресурсы
- Система баланса слушает событие → обновляет баланс

**Преимущества:**
- Нет прямых зависимостей
- Легко добавлять новые модули
- Просто тестировать и отлаживать

## 🛠️ Системы и модули

### 1. Core (Основные системы)

#### 1.1. Система авторизации (Authorization)
**Описание:** Управление аккаунтами игроков.

**Компоненты:**
- **Authentication** — вход в аккаунт
- **Authorization** — проверка прав
- **Session Management** — управление сессией

**Технические детали:**
- Использует JWT (JSON Web Token)
- Сессии хранятся в Redis
- База данных: PostgreSQL

#### 1.2. Система сохранения (Save System)
**Описание:** Сохранение прогресса игрока.

**Компоненты:**
- **Local Save** — локальное сохранение
- **Cloud Save** — облачное сохранение
- **Sync System** — синхронизация

**Технические детали:**
- Сохранение в JSON
- Синхронизация через REST API
- База данных: MongoDB

### 2. Gameplay (Игровые механики)

#### 2.1. Система строительства (Building System)
**Описание:** Строительство баз, укреплений и сооружений.

**Компоненты:**
- **Grid Snapping** — сетка привязки
- **Block System** — блоки и их свойства
- **Damage System** — разрушение блоков

**Технические детали:**
- Использует Grid-based system
- Физика: Unity Physics
- Сохранение: массив блоков с позицией

#### 2.2. Система боя (Combat System)
**Описание:** Боевая система с учётом баллистики, укрытий.

**Компоненты:**
- **Ballistics Calculator** — расчёт траектории пули
- **Damage System** — расчёт урона по зонам
- **Cover System** — укрытия и позиционирование

**Технические детали:**
- Гибридная система: Raycast + Projectile
- Пул объектов для пуль
- LOD для моделей оружия

### 3. Economy (Экономика)

#### 3.1. Система валют (Currency System)
**Описание:** Двойная валютная система (Койны и Золото).

**Компоненты:**
- **Currency Manager** — управление валютами
- **Transaction System** — транзакции
- **Exchange System** — биржа и аукцион

**Технические детали:**
- Сериализация в JSON
- Транзакции: MongoDB
- Синхронизация: REST API

#### 3.2. Система биржи (Exchange System)
**Описание:** Биржа и аукцион для торговли.

**Компоненты:**
- **Auction Manager** — аукцион
- **Exchange Manager** — биржа
- **Trade System** — торговая система

**Технические детали:**
- Транзакции: MongoDB
- Синхронизация: REST API
- Кэширование: Redis

### 4. UI (Интерфейс)

#### 4.1. Система HUD (HUD System)
**Описание:** Интерфейс игрока (здоровье, инвентарь).

**Компоненты:**
- **Health Bar** — полоса здоровья
- **Inventory UI** — инвентарь
- **Crafting UI** — интерфейс крафта

**Технические детали:**
- Использует Unity UI
- Кэширование: Object Pooling
- Оптимизация: Canvas Group

#### 4.2. Система меню (Menu System)
**Описание:** Меню игры (главное меню, настройки).

**Компоненты:**
- **Main Menu** — главное меню
- **Settings Menu** — настройки
- **Pause Menu** — пауза

**Технические детали:**
- Использует Unity UI
- Анимация: Unity Animator
- Оптимизация: Canvas Group

### 5. Networking (Сеть)

#### 5.1. Система сетевого кода (Netcode System)
**Описание:** Сетевой код для синхронизации игроков.

**Компоненты:**
- **Netcode for GameObjects** — основа
- **Sync System** — синхронизация состояния
- **Prediction System** — предсказание

**Технические детали:**
- Использует Netcode for GameObjects
- Синхронизация: UDP
- Предсказание: клиент-серверная модель

#### 5.2. Система лобби (Lobby System)
**Описание:** Лобби для создания и присоединения к играм.

**Компоненты:**
- **Lobby Manager** — управление лобби
- **Matchmaking** — подбор игроков
- **Room System** — комнаты

**Технические детали:**
- Использует PlayFab
- Синхронизация: REST API
- База данных: PostgreSQL

### 6. Database (База данных)

#### 6.1. Система хранения (Storage System)
**Описание:** Хранение данных игроков и предметов.

**Компоненты:**
- **Player Data** — данные игрока
- **Item Data** — данные предметов
- **World Data** — данные мира

**Технические детали:**
- База данных: MongoDB
- Кэширование: Redis
- Синхронизация: REST API

#### 6.2. Система баланса (Balance System)
**Описание:** Баланс экономики и ресурсов.

**Компоненты:**
- **Currency Balance** — баланс валют
- **Resource Balance** — баланс ресурсов
- **Transaction Log** — лог транзакций

**Технические детали:**
- База данных: PostgreSQL
- Транзакции: MongoDB
- Кэширование: Redis
